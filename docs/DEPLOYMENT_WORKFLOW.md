# Deployment Workflow for TwisterLab v5.0.10\n\n## Overview\nThis document outlines the complete deployment process for TwisterLab v5.0.10, covering preparation, execution, verification, and rollback procedures.\n\n---\n\n## Pre-Deployment Checklist\n\n### 1. Code & Artifact Preparation\n- [ ] Build Docker image: `docker build -t twisterlab-api:v5.0.10`\n- [ ] Export PostgreSQL data: `docker exec postgres -U twisterlab pg_dump -F\'c\' twisterlab > backup.sql`\n- [ ] Export Redis data (if needed): `docker exec redis -U twisterlab redis-cli SAVE`\n- [ ] Verify container images are up-to-date\n\n### 2. Configuration Verification\n- [ ] Confirm environment variables are set:\n  - `POSTGRES_PASSWORD` (required)\n  - `REDIS_PASSWORD` (required)\n  - `GRAFANA_PASSWORD` (required)\n- [ ] Validate Kubernetes resource limits:\n  ```yaml\n  deploy:\n    resources:\n      limits:\n        cpus: \'1\'\n        memory: 1G\n  ```\n\n### 3. Dependency Checks\n- [ ] Verify required tools are available:\n  - Docker (>=20.10)\n  - kubectl (>=v1.19)\n  - nginx (for reverse proxy)\n  - Prometheus & Grafana (for monitoring)\n\n---\n\n## Deployment Steps\n\n### Step 1: Build and Transfer\n```bash\n# Pull latest images\ndocker pull ghcr.io/youneselfakir0/twisterlab/api:v5.0.10\ndocker pull ghcr.io/youneselfakir0/twisterlab/mcp:latest\n\n# Build and transfer\nkubectl apply -f k8s/deployments/twisterlab-api.yaml -n twisterlab\ndocker exec -it --entrypoint /bin/bash twisterlab-api -c "docker save -o api.tar $Dockerfile"\nscp api.tar user@remote-host:/tmp/api.tar.gz\n```\n\n### Step 2: Apply Configuration\n```bash\n# Set environment variables from .env.production.example\ncp .env.production.example .env\n# Modify required values in .env\n```\n\n### Step 3: Deploy to Kubernetes\n```bash\nkubectl apply -f k8s/deployments/twisterlab-api.yaml -n twisterlab\nkubectl rollout status deployment/twisterlab-api -n twisterlab\n```\n\n### Step 4: Initial Verification\n```bash\n# Check API health\ncurl http://localhost/health\n\n# Test core endpoints\ncurl -X POST http://localhost/v1/mcp/tools/list_autonomous_agents\ncurl -X POST http://localhost/v1/mcp/tools/analyze_sentiment -H "Content-Type: application/json" -d \'{"text":"test","detailed":true}\'\n```\n\n---\n\n## Post-Deployment Validation\n\n### System Health Checks\n- [ ] API responds with health status\n- [ ] Agents can connect to backend services\n- [ ] Database connections are healthy\n- [ ] Monitoring agents are operational\n\n### Data Integrity Verification\n- [ ] Run migration scripts: `./scripts/migrate_data_v2.py`\n- [ ] Verify data consistency between systems\n- [ ] Confirm audit logs are being collected\n\n### Performance Baseline Measurement\n- [ ] Measure API response times\n- [ ] Monitor resource utilization (CPU, memory)\n- [ ] Validate database query performance\n\n---\n\n## Rollback Procedures\n\n### Quick Rollback (Last Deployment)\n```bash\nkubectl rollout undo deployment/twisterlab-api -n twisterlab\nkubectl scale deployment/twisterlab-api --replicas=3 -n twisterlab\n```\n\n### Full Rollback with Database Restore\n```bash\n# Stop and remove old deployment\nkubectl delete -f k8s/deployments/twisterlab-api.yaml -n twisterlab\nkubectl scale deployment/twisterlab-api --replicas=0 -n twisterlab\n\n# Restore from backup\nkubectl exec -it --entrypoint /bin/bash postgres -n twisterlab -c "psql -U twisterlab -f backup_$(date +%Y%m%d).sql"\n\n# Recreate deployment\nkubectl apply -f k8s/deployments/twisterlab-api.yaml -n twisterlab\n```\n\n---\n\n## Configuration Reference\n\n### Environment Variables (from .env.production.example)\n- `POSTGRES_PASSWORD` - Database password (must be changed from default)\n- `REDIS_PASSWORD` - Redis password (must be changed from default)\n- `GRAFANA_PASSWORD` - Grafana admin password (must be changed from default)\n\n### Image Tags Used in Deployment\n- API image: `ghcr.io/youneselfakir0/twisterlab/api:v5.0.10`\n- MCP agent: `ghcr.io/youneselfakir0/twisterlab/mcp:latest`\n\n### Secrets Management\n- All secrets stored in Kubernetes Secret objects\n- Access controlled via RBAC policies\n- No hardcoded credentials in deployment scripts\n\n---\n\n## Operational Notes\n- Deployments typically complete within 5-10 minutes\n- Maximum resource usage: 1 CPU core, 1GB RAM per replica\n- Health checks performed every 5 minutes during rollout\n- All critical changes require manual approval\n
